# 一、观察者模式
观察者模式，又称发布订阅模式，是一种行为设计模式——你可以定义一种订阅机制，可在对象事件发生时通知多个 观察 该对象的其他对象。
  - 被观察者 (Observable) 或发布者 (Subject)，例如：【Flutter社区】公众号
  - 观察者，订阅 【Flutter社区】公众号的用户。

观察者模式定义了一种一对多的依赖关系，让多个观察者对象同时监听某一个主题对象。
这个主题对象在状态上发生变化时，会通知所有观察者对象，让它们能够自动更新自己。


# Listenable 
<https://api.flutter.dev/flutter/foundation/Listenable-class.html>

Listenable 是一个核心抽象类，位于 foundation.dart 中，用于实现可监听对象。
Listenable 是一个维护 listeners(监听器) 列表的对象，监听器是一个回调函数，当 Listenable 对象的状态发生变化时被调用。

Listenable 有2种增强的接口：
  - Animation
  - ValueListenable‌

Flutter API 中的许多类都实现了 Listenable 接口，例如：
  - ChangeNotifier，通过被子类化或混合使用，以创建实现 Listenable 接口的对象。
  = ValueNotifier，其包裹的可变值会在修改时触发通知。


## 观察者 Widget

* AnimatedBuilder，它是 AnimatedWidget 的子类，可用于创建由 Listenable 驱动的小部件。

* ValueListenableBuilder，它将自动将自己注册为 ValueListenable 的监听器，并在值更改时使用更新后的值调用构建器。

* InheritedNotifier，一个抽象超类，用于使用 Listenable 的通知来触发依赖于它们的后代小部件的重建，使用 InheritedWidget 机制。
